# MidShake Roadmap

This roadmap outlines the planned direction for MidShake as the language and toolchain mature. The focus is on making the language more expressive, easier to use, and better suited for real programs.

## Near-term priorities

### 1. Language polish

- improve parser reliability for edge cases
- refine error messages so they are clearer and more actionable
- expand support for common expressions and value forms

### 2. Standard library growth

- add string helpers such as trim, join, split, and case conversion
- add math helpers such as round, min, max, and power
- add list and map support with basic collection operations
- add more input and output convenience functions

### 3. Tooling and developer experience

- improve CLI commands and documentation
- add better examples for common workflows
- support an interactive REPL for quick experimentation
- improve packaging and release workflow

## Medium-term goals

### 4. Modules and code organization

- support importing external MidShake files
- introduce a simple module system for reusable code
- make libraries easier to share and compose

### 5. Compiler and runtime improvements

- add a compiler mode that turns MidShake into native or optimized output
- improve runtime performance for loops and function calls
- strengthen compatibility between the Python and C++ backends

### 6. Better ecosystem support

- formalize the language syntax more clearly
- provide stronger examples and reference material
- add a formal test suite for language behavior and regressions

## Long-term vision

MidShake should become a friendly, readable language for teaching, prototyping, and small automation tasks. Over time, it can grow from a lightweight interpreter into a more complete language platform with better tooling, stronger libraries, and clearer documentation.

## Suggested milestone order

1. parser and error-message improvements
2. standard library expansion
3. REPL and interactive workflow
4. modules and import support
5. compiler pipeline and performance work
6. larger ecosystem and tooling maturity
